# Skill Pilot 手动安装指南（macOS 与 Linux）

如果 `install.sh` 或 `skillpilot.sh` 的初始化流程无法自动完成，请使用本指南。

## 0. 前置条件

- 支持的操作系统：macOS 或 Linux（Windows 用户应先使用 WSL）。
- 至少准备好以下其中一项：
1. Claude Code、GitHub Copilot CLI、Codex、Gemini CLI 或 OpenCode CLI
2. 兼容 OpenAI 或兼容 Claude 的 API 地址与密钥

如果你既没有 agent CLI，也没有 API 凭据，那么安装将无法完成。

## 1. 安装 Homebrew（如果尚未安装）

`install.sh` 以 Homebrew 作为基础包管理器。

```bash
NONINTERACTIVE=1 /bin/bash -c "$(curl -fsSL --proto '=https' --tlsv1.2 https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

安装完成后，将 Homebrew 加载到当前 shell：

Apple Silicon macOS：

```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```

Intel macOS：

```bash
eval "$(/usr/local/bin/brew shellenv)"
```

Linuxbrew：

```bash
eval "$(/home/linuxbrew/.linuxbrew/bin/brew shellenv)"
```

## 2. 安装基础工具与运行时

使用 Homebrew 安装基础软件包：

```bash
brew install git curl tmux
```

使用官方脚本安装 `uv`（与 `install.sh` 相同的方法）：

```bash
curl -fsSL --proto '=https' --tlsv1.2 https://astral.sh/uv/install.sh | sh
```

使用官方脚本安装 `pnpm`（与 `install.sh` 相同的方法）：

```bash
curl -fsSL --proto '=https' --tlsv1.2 https://get.pnpm.io/install.sh | sh
```

设置 `PNPM_HOME` 并更新 `PATH`：

macOS：

```bash
export PNPM_HOME="${PNPM_HOME:-$HOME/Library/pnpm}"
export PATH="$PNPM_HOME:$PATH"
```

Linux：

```bash
export PNPM_HOME="${PNPM_HOME:-$HOME/.local/share/pnpm}"
export PATH="$PNPM_HOME:$PATH"
```

安装 Node.js LTS（要求：Node 18+）：

```bash
pnpm env use --global lts
```

安装 Python 3.9+（如果缺失）：

```bash
uv python install
```

安装 agent-browser CLI：

```bash
pnpm install -g agent-browser
```

## 3. 至少安装一个 AI 代码代理 CLI

`skillpilot.sh` 会检查以下受支持的提供方：

- `claude`
- `copilot`
- `codex`
- `gemini`
- `opencode`

初始化流程中使用的安装命令如下：

```bash
curl -fsSL https://claude.ai/install.sh | bash
pnpm install -g @github/copilot
pnpm install -g @openai/codex
pnpm install -g @google/gemini-cli
pnpm install -g opencode-ai
```

说明：

- `pnpm install -g ...` 需要 `pnpm` 和可正常工作的 Node.js 安装。
- 使用以下命令验证 CLI 是否已安装：

```bash
command -v claude
command -v copilot
command -v codex
command -v gemini
command -v opencode
```

## 4. 克隆仓库并设置分支

克隆完成后，默认分支是 `codeware`（稳定发布层）。
在开始修改之前，请创建本地工作分支并切换到 `user`。

简要含义：
- `codeware`：原始软件基线（稳定上游分支）
- `contrib`：用于发起 pull request 共享改进的分支
- `user`：你个人的工作区分支，用于日常 AI / 用户编辑

```bash
git clone https://github.com/x-school-academy/skill-pilot "$HOME/workspace/skill-pilot"
cd "$HOME/workspace/skill-pilot"

# clone 后的默认分支：codeware
git branch contrib
git branch user
git checkout user
```

如果 `contrib` 或 `user` 已存在，则跳过对应分支的创建。

## 5. 安装项目依赖

在仓库根目录执行：

```bash
uv --project core/engine sync
pnpm -C core/webui install
```

## 6. 创建 `config/.env`

至少创建包含以下内容的 `config/.env`：

```dotenv
ONLY_ALLOW_HTTPS=0
AUTH_TOKEN=<uuid-or-random-secret>
```

生成 UUID token：

```bash
core/engine/.venv/bin/python - <<'PY'
import uuid
print(uuid.uuid4())
PY
```

为你选择的提供方添加凭据（如果需要）。`config/ai_providers.json5` 中的兼容 API 提供方和 Ollama 提供方会直接读取这些环境变量：

面向 `codex-compat` 的 OpenAI 兼容配置：

```dotenv
OPENAI_COMPAT_BASE_URL=https://your-openai-compatible-endpoint
OPENAI_COMPAT_API_KEY=<your-openai-key>
```

面向 `claude-compat` 的 Claude 兼容配置：

```dotenv
ANTHROPIC_COMPAT_BASE_URL=https://your-claude-compatible-endpoint
ANTHROPIC_COMPAT_AUTH_TOKEN=<your-claude-token>
ANTHROPIC_API_KEY=
```

Ollama：

```dotenv
# 默认已在 config/ai_providers.json5 中指向本地 Ollama
# 只有当你的 Ollama 地址不同的时候才需要修改。
OPENAI_BASE_URL=http://localhost:11434/v1/
OPENAI_API_KEY=ollama
```

推荐权限：

```bash
chmod 600 config/.env
```

## 7. 更新 `config/settings.json5`

设置主机与服务端口（示例）：

```json5
{
  "services": {
    "engine": {
      "host": "127.0.0.1",
      "production": {
        "port": 3001
      },
      "development": {
        "port": 3002
      }
    },
    "webui": {
      "host": "127.0.0.1",
      "development": {
        "port": 3003
      }
    }
  }
}
```

如果要绑定到非 localhost 主机，请决定是否强制启用 HTTPS（`ONLY_ALLOW_HTTPS=1`）或者允许 HTTP（`ONLY_ALLOW_HTTPS=0`）。

端口含义：

- `engine.production.port`：生产环境 engine 和打包后的发布版 WebUI，默认 `3001`
- `engine.development.port`：开发环境 engine，默认 `3002`
- `webui.development.port`：Next.js 开发环境 WebUI，默认 `3003`

## 8. 更新 `config/ai_providers.json5`

- 将 `default.llm` 设置为你选择的提供方 id：
  - 原生 CLI 模式：`claude`、`copilot`、`codex`、`gemini` 或 `opencode`
  - 兼容 API 模式：`claude-compat` 或 `codex-compat`
  - 本地 Ollama 模式：`ollama`
- 对未使用的 LLM 提供方设置 `disabled: true`。
- 保持你选择的提供方启用（`disabled: false` 或省略 `disabled`）。
- 内置的兼容提供方默认是禁用的。若要启用，请编辑其在 `llm` 下的条目，并删除 `disabled: true` 或把它改成 `disabled: false`。

选择示例：

```json5
default: {
  llm: 'ollama',
  tts: 'openai',
  image: 'openai',
}
```

```json5
{
  id: 'ollama',
  disabled: false,
  model: 'llama3.2',
  env: {
    OPENAI_API_KEY: 'ollama',
    OPENAI_BASE_URL: 'http://localhost:11434/v1/',
  },
}
```

## 9. 启动 Skill Pilot

生产模式：

```bash
./skillpilot.sh
```

开发模式：

```bash
./skillpilot.sh --dev
```

同时运行两者：

```bash
./skillpilot.sh
./skillpilot.sh --dev
```

仅停止生产模式：

```bash
./skillpilot.sh stop
```

仅停止开发模式：

```bash
./skillpilot.sh stop --dev
```

## 10. 直接使用 AI Agent CLI

如果你直接使用 AI agent CLI，而不使用 Skill Pilot WebUI：

- 通过运行 `./skillpilot.sh` 使用生产模式启动 engine
- 开发时，在直接启动 agent CLI 之前先导出 `SKILL_PILOT_RUNTIME_MODE=development`

示例：

```bash
export SKILL_PILOT_RUNTIME_MODE=development
claude
```

```bash
export SKILL_PILOT_RUNTIME_MODE=development
codex
```

```bash
export SKILL_PILOT_RUNTIME_MODE=development
gemini
```

相同规则也适用于其他受支持的 agent CLI，例如 Copilot CLI 或 OpenCode。

```json5
{
  id: 'codex-compat',
  disabled: false,
  model: 'your-openai-compatible-model',
}
```

```json5
{
  id: 'claude-compat',
  disabled: false,
  model: 'your-claude-compatible-model',
}
```

提供方说明：

- `claude-compat` 读取 `ANTHROPIC_COMPAT_BASE_URL` 和 `ANTHROPIC_COMPAT_AUTH_TOKEN`。
- `codex-compat` 读取 `OPENAI_COMPAT_BASE_URL` 和 `OPENAI_COMPAT_API_KEY`。
- `ollama` 已预配置为通过 Codex CLI 适配器使用 `http://localhost:11434/v1/`；请把它的 `model` 更新成你实际安装的 Ollama 模型。
- `opencode` 也可以通过编辑 `config/ai_providers.json5` 中的 `env` 块，指向一个兼容 OpenAI 的端点。

## 9. 启动服务

开发模式：

```bash
./skillpilot.sh start --dev
```

生产模式：

```bash
./skillpilot.sh start
```

停止服务：

```bash
./skillpilot.sh stop
```

## 10. 可选依赖包

仅在需要时启用可选功能：

```bash
./skillpilot.sh enable human-detection
./skillpilot.sh enable live-tts
```

`enable live-tts` 会在 macOS / Linux 上安装音频构建依赖（PortAudio + pkg-config），然后安装 `core/engine/mcp_servers/live_tts/requirements-live-tts.txt` 中列出的 Python 包。

禁用可选包：

```bash
./skillpilot.sh disable human-detection
./skillpilot.sh disable live-tts
```

## 说明

- 当 `config/.env` 缺失时，`skillpilot.sh` 会自动运行交互式初始化向导。
- 在可用时，向导会使用 `core/engine/.venv` 中的 Python 来更新 `config/.env`、`config/settings.json5` 和 `config/ai_providers.json5`。
