## 什么是 Skill Pilot？

Skill Pilot 是一个您在本地运行的 **AI 原生工作空间**。它将您喜爱的 AI 编程智能体（Claude Code、GitHub Copilot CLI、Codex、Gemini CLI 等）连接到一个由可复用技能、工具和工作流组成的结构化环境。

不同于将 AI 用作聊天助手，Skill Pilot 将 AI 转变为一个主动的工作者，它可以：

- 读取和修改您的代码库
- 在您的电脑上执行任务
- 通过可复用的"技能"学习新能力
- 在您的整个数字工作流程中运作 —— 不仅仅是代码

**简而言之：** 您描述您想要的。AI 去做。系统每次都变得更好。

---

## 什么是代码件（Codeware）？

传统软件是静态的 —— 它带有固定的功能，只有当开发者更新时才会改变。

**代码件不同。** 源代码本身就是产品，AI 根据您的需求持续扩展它。

```
传统方式：  人类 → 界面 → 软件 → 固定行为
代码件：    人类 → AI → 代码库 → 进化行为
```

在 Skill Pilot 中：

| 传统软件 | Skill Pilot 代码件 |
|---------|-------------------|
| 您使用功能 | 您培养能力 |
| 更新来自开发者 | 更新来自 AI + 您 |
| 发布时固定 | 通过使用进化 |
| 一刀切 | 适应您的需求 |

系统不是被"使用"的。它是被**培养**的。

## 使用方法

[安装](#安装)后，运行：

```bash
# cd <您的安装路径>
./skillpilot.sh help
```

```
用法: ./skillpilot.sh [help|build|start|stop|test] [--dev]
      ./skillpilot.sh <enable|disable> <human-detection|live-tts>

命令:
  help    显示此帮助信息。
  build   构建静态 webui 导出 (core/webui/www)。
  start   启动服务。默认命令。
  stop    停止运行中的 tmux 会话。使用 `--dev` 仅停止开发会话。
  test    运行引擎 pytest 测试套件，或仅运行 core/engine/tests 下的指定文件。
  enable human-detection    安装可选的人体检测依赖。
  disable human-detection   卸载可选的人体检测依赖。
  enable live-tts           安装可选的实时语音合成依赖。
  disable live-tts          卸载可选的实时语音合成依赖。

选项:
  --dev   为 `start` 使用开发模式，或为 `stop` 仅停止开发会话。

默认值:
  - 默认命令为: start
  - 默认模式为生产模式（不带 --dev）
```

### 生产和开发模式

Skill Pilot 现在支持同时运行生产和开发环境：

- 生产引擎和打包的发布 WebUI：`http://127.0.0.1:3001`
- 开发引擎：`http://127.0.0.1:3002`
- 开发 WebUI：`http://127.0.0.1:3003`

典型工作流：

```bash
./skillpilot.sh
./skillpilot.sh --dev
```

停止命令：

```bash
./skillpilot.sh stop
./skillpilot.sh stop --dev
```

- `./skillpilot.sh stop` 仅停止生产环境
- `./skillpilot.sh stop --dev` 仅停止开发环境

### 直接使用 AI 智能体 CLI

如果您直接使用智能体 CLI，而不通过 Skill Pilot WebUI：

- 对于正常使用，用 `./skillpilot.sh` 以生产模式启动引擎
- 对于开发工作流，在直接启动智能体 CLI 之前导出 `SKILL_PILOT_RUNTIME_MODE=development`

示例：

```bash
export SKILL_PILOT_RUNTIME_MODE=development
claude
```

```bash
export SKILL_PILOT_RUNTIME_MODE=development
codex
```

```bash
export SKILL_PILOT_RUNTIME_MODE=development
gemini
```

这确保智能体 CLI 可以看到与开发引擎相同的运行时模式。

## 您的工作空间

安装后，您的工作空间被组织成专注的区域 —— 每个都由 Skill Pilot 驱动。

```
skill-pilot/
├── src/                    ← 您的商业级产品
│   └── ...                    由 Dev Swarm 智能体端到端构建
│
└── workspace/
    ├── learning/           ← 您的 AI 学习工作室
    │   └── ...                课程、教程和交互式课程
    │
    ├── projects/           ← 您的氛围编程实验室
    │   └── ...                快速启动任何编程项目
    │
    ├── research/           ← 您的研究工作空间
    │   └── ...                深度调研、分析和知识收集
    │
    └── tasks/              ← 您的 AI 任务运行器
        └── ...                用 AI 自动化任何非编程任务
```

| 区域 | 是什么 | 最适合 |
|------|--------|--------|
| `src/` | 您正在构建的产品 —— 从想法到部署，由 Dev Swarm 管理 | 企业、产品构建者 |
| `workspace/learning/` | AI 驱动的课程、教程和技能建设 | 初学者、学习者 |
| `workspace/projects/` | 氛围编程空间 —— 描述您想要的，看着它被构建 | 初学者、开发者 |
| `workspace/research/` | 分析、竞争研究和知识综合的研究工作空间 | 专业人士、研究人员 |
| `workspace/tasks/` | 处理其他一切的任务运行器 —— 写作、自动化、规划等 | 专业人士、所有人 |

## Skill Pilot 如何工作

1. 您在本地机器上运行 Skill Pilot
2. AI 读取您的仓库并理解您的项目
3. 您用自然语言描述您想要的
4. AI 编辑代码、运行任务或创建新技能来解决它
5. 新能力被保存为可复用技能供下次使用
6. 有用的技能可以通过 `contrib` 分享回社区
7. 稳定、经过验证的技能被发布到 `codeware` 供所有人使用

**系统每次使用都在改进。**

---

## 分支模型

Skill Pilot 将三层演进分开，这样您的个人工作永远不会与共享发布冲突。

| 分支 | 目的 | 谁编写 |
|------|------|--------|
| `codeware` | 稳定发布层 | 维护者 |
| `contrib` | 用户之间共享的改进 | 社区 + AI 审查 |
| `user` | 您的个人进化工作空间 | 您 + 您的 AI |

### 知识流动

```
codeware  → user        （接收更新）
user      → contrib     （分享有效的内容）
contrib   → codeware    （稳定发布）
```

规则：

- `codeware` 是受保护的 —— 仅发布
- `contrib` 接受来自社区的拉取请求
- `user` 由您和您的 AI 自由修改
- AI 永远不会直接编辑 `codeware`

---

## 贡献

您不是通过为用户编写功能来贡献。您是通过**教系统一个可复用的技能**来贡献。

步骤：

1. 在您的 `user` 分支中工作
2. 如果多个用户共享一台电脑，每个用户应该 fork 到不同的仓库并克隆到本地不同的路径
3. 当某些东西有用时 → 创建一个到 `contrib` 的 PR
4. 维护者和 AI 审查集成安全性
5. 合并到共享知识中
6. 发布到 `codeware`

## 理念

> 软件不应该包含所有知识。它应该学习知识。

传统软件分发功能。
代码件分发能力进化。

您不是在安装一个应用程序。

您是在安装**一个 AI 工作者的栖息地**。

AI 不是在程序内部运行。
程序是在 AI 内部运行。
