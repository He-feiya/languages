# Skill Pilot 安装配置说明

`config` 目录是 Skill Pilot 的核心配置中心，主要用于管理环境变量、服务端口、AI 提供方、MCP 服务器、SSH 连接以及权限策略。

## 配置目录文件总览

| 文件 | 作用 |
| --- | --- |
| `config/.env.example` | 环境变量示例文件，列出运行时可能需要的密钥和开关。 |
| `config/ai_providers.json5` | AI 提供方配置，定义 LLM、TTS、图像生成等提供方及默认选择。 |
| `config/claude-sandbox-settings.json` | Claude Code 的沙箱与网络访问设置。 |
| `config/disabled_skills.json5` | 显式禁用的技能列表。 |
| `config/mcp.json5` | MCP 服务器注册表，定义本地与远程 MCP 服务。 |
| `config/mcp_descriptions.yaml` | MCP 技能描述覆盖文件，用于替换默认说明。 |
| `config/opencode-yolo.json` | OpenCode 的权限配置文件。 |
| `config/settings.json5` | 服务端口、安全策略、WebUI / engine 地址等基础运行配置。 |
| `config/ssh.json5` | 实际使用的 SSH 连接配置。 |
| `config/ssh.template.json5` | SSH 配置模板。 |

## 1. `config/.env.example`

这是环境变量模板，主要用于提示你哪些变量可能需要填写。

当前示例里包含的项目有：

```dotenv
AUTH_TOKEN=
IN_KEYS_SAFE_GUARD=1
OLLAMA_API_KEY=
OPENROUTER_API_KEY=
PLAYWRIGHT_MCP_EXTENSION_TOKEN=
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_REGION=
DISABLE_AWS_API_MCP_SERVER=false
```

常见用途：

- `AUTH_TOKEN`：WebUI 或接口访问令牌。
- `OPENROUTER_API_KEY`、`OLLAMA_API_KEY`：不同 AI 提供方所需密钥。
- `PLAYWRIGHT_MCP_EXTENSION_TOKEN`：Playwright 扩展相关认证。
- `AWS_*`：AWS API MCP 服务所需凭据。
- `DISABLE_AWS_API_MCP_SERVER`：控制是否禁用 AWS API MCP 服务。

## 2. `config/settings.json5`

这个文件负责定义 Skill Pilot 的基础运行方式，包括安全策略、服务监听地址、端口和 TURN 配置。

### 安全策略

`security` 段定义不同场景下的沙箱、自动执行和网络访问策略：

```json5
security: {
  schedules: { sandbox: true, auto: true, network: true },
  newSession: { sandbox: false, auto: false, network: true },
  remoteBot: { sandbox: true, auto: true, network: false },
  devSwarm: { sandbox: true, auto: true, network: true },
  skillAgent: { sandbox: true, auto: true, network: true },
}
```

### 服务端口

`services` 段定义 engine 和 WebUI 的地址与端口：

```json5
services: {
  engine: {
    host: "127.0.0.1",
    production: { port: 3001 },
    development: { port: 3002 },
  },
  webui: {
    host: "127.0.0.1",
    development: { port: 3003 },
  },
}
```

含义如下：

- `engine.production.port`：生产环境 engine 端口，默认 `3001`
- `engine.development.port`：开发环境 engine 端口，默认 `3002`
- `webui.development.port`：开发环境 WebUI 端口，默认 `3003`

### Live Avatar 与 TURN

文件中还包含：

- `live_avatar.server_url`：Live Avatar 的 WebSocket 地址
- `turn.urls`、`turn.username`、`turn.password`：TURN 服务器配置

这些项主要服务于摄像头 MCP 和 live-avatar 功能。

## 3. `config/ai_providers.json5`

这是最重要的 AI 提供方配置文件之一，用于定义：

- 默认 LLM / TTS / 图像提供方
- 各 CLI 提供方的启动命令
- 传入模型参数
- 沙箱参数、自动批准参数、联网参数
- 环境变量映射
- 是否禁用某个提供方

### 默认提供方

```json5
default: {
  llm: 'claude',
  tts: 'media-mcp',
  image: 'media-mcp',
}
```

### 已定义的 LLM 提供方

文件中可见的主要 LLM 提供方包括：

- `claude`
- `copilot`
- `codex`
- `gemini`
- `opencode`
- `claude-compat`
- `claude-openrouter`
- `codex-compat`
- `ollama-cloud`

### 配置字段含义

每个提供方通常包含以下字段：

- `id`：唯一标识
- `name`：显示名称
- `bin`：实际执行的命令
- `model`：模型名称
- `args`：命令参数，通常带有 `{{prompt}}`
- `sandbox-args`：启用沙箱时附加的参数
- `auto-args`：启用自动批准时附加的参数
- `network-args`：启用网络时附加的参数
- `env`：传递给进程的额外环境变量
- `disabled`：是否禁用

### 示例：Claude

```json5
{
  id: 'claude',
  name: 'Claude Code',
  bin: 'claude',
  'sandbox-args': ['--settings', 'claude-sandbox-settings.json'],
  'auto-args': ['--dangerously-skip-permissions'],
  args: [
    '--include-partial-messages',
    '--output-format=stream-json',
    '--verbose',
    '--print',
    '{{prompt}}',
  ],
}
```

### 示例：Copilot

```json5
{
  id: 'copilot',
  name: 'GitHub Copilot CLI',
  bin: 'copilot',
  'auto-args': ['--no-ask-user'],
  'network-args': ['--allow-all-urls'],
}
```

### 示例：兼容接口和 Ollama

这个文件还支持：

- 兼容 Claude 的 API 端点（`claude-compat`）
- 兼容 OpenAI 的 API 端点（`codex-compat`）
- OpenRouter
- Ollama / 本地兼容接口

也就是说，除了原生 CLI 外，Skill Pilot 还能通过环境变量连接第三方兼容接口。

## 4. `config/mcp.json5`

这个文件定义了 MCP 服务器注册表。

当前配置中包含的 MCP 服务示例：

- `terminal`
- `media`
- `live-tts`
- `cameras`
- `dart`
- `context7`
- `aws-api`
- `github`

### 本地 Python MCP 服务

例如：

```json5
terminal: {
  system: true,
  command: "uv",
  args: [
    "--directory",
    "core/engine",
    "run",
    "mcp_servers/terminal/main.py",
  ],
}
```

这表示通过 `uv` 在 `core/engine` 下运行对应的 MCP Python 服务。

### Media MCP

`media` 服务包含较多环境变量，例如：

- `COMFYUI_SERVER_ADDRESS`
- `COMFYUI_INSTALL_PATH`
- `GPU_WORKER_TTS_CREATOR`
- `GPU_WORKER_SONG_CREATOR`
- `GPU_WORKER_WHISPER_CLI`

这说明 `media` MCP 会连接外部多媒体与 GPU 工具链。

### HTTP 型 MCP

例如：

```json5
context7: {
  type: "http",
  url: "https://mcp.context7.com/mcp",
}
```

以及：

```json5
github: {
  type: "http",
  url: "https://api.githubcopilot.com/mcp/",
  headers: {
    Authorization: "Bearer ${GITHUB_MCP_PAT}",
    "X-MCP-Toolsets": "actions",
  },
  disabled: true,
}
```

这类服务依赖远程 MCP 端点和认证头。

## 5. `config/mcp_descriptions.yaml`

这是 MCP 技能描述覆盖文件。

它的作用是：

- 用自定义说明替换 MCP 工具默认生成的描述
- 让技能名称和说明更清晰、更适合最终展示

示例：

```yaml
context7-resolve-library-id: >
  Resolve a package or product name to the best matching Context7 library ID before calling query-docs.
```

对应中文意思就是：在调用 `query-docs` 之前，先把包名或产品名解析为最匹配的 Context7 library ID。

## 6. `config/disabled_skills.json5`

这个文件列出了需要显式禁用的技能。

当前示例包括：

```json5
[
  "cameras-webrtc-ice-candidate",
  "cameras-webrtc-offer",
  "context7-query-docs",
  "context7-resolve-library-id",
  "media-image-analysis",
  "media-video-analysis"
]
```

用途是：

- 在系统层面关闭某些技能
- 防止不稳定、无权限或不需要的功能出现在可用列表中

## 7. `config/claude-sandbox-settings.json`

这个文件是 Claude Code 的沙箱设置。

示例：

```json
{
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": true,
    "allowUnsandboxedCommands": true,
    "network": {
      "allowedDomains": ["*"],
      "allowAllUnixSockets": true
    }
  }
}
```

核心含义：

- 启用沙箱模式
- 在沙箱中自动允许 Bash
- 允许未沙箱化命令
- 允许访问全部域名和 Unix Socket

## 8. `config/opencode-yolo.json`

这是 OpenCode 的权限配置文件。

从内容看，它定义了：

- `bash` 权限
- `edit` 权限
- `read` 权限
- `webfetch` / `websearch` 权限

示例：

```json
"read": {
  "*": "allow",
  "config/.env": "deny"
}
```

说明：

- 默认允许读取多数文件
- 但明确禁止读取 `config/.env`

这是一种很重要的安全设计，用来避免敏感配置被直接读出。

## 9. `config/ssh.template.json5` 与 `config/ssh.json5`

这两个文件用于 SSH 连接配置。

### 模板文件

`ssh.template.json5` 提供了标准结构：

```json5
{
  "profiles": {
    "prod": {
      "host": "1.2.3.4",
      "port": 22,
      "user": "deploy",
      "password": "",
      "key": "~/.ssh/id_rsa",
      "knownHosts": "~/.ssh/known_hosts",
      "timeoutMs": 60000,
      "maxChars": 1000,
      "sudoPassword": ""
    }
  }
}
```

### 实际文件

`ssh.json5` 则存放实际环境中的连接配置，例如：

```json5
profiles: {
  "ec2-tmp": {
    host: "54.79.12.72",
    port: 22,
    user: "ubuntu",
    key: ".skillpilot/temp/ec2-tmp-ssh",
    timeoutMs: 60000,
  },
}
```

这说明 Skill Pilot 支持通过配置文件保存远程主机连接信息。

## 安装与配置时最常需要修改的文件

如果你是首次安装，通常最常改的是这几个文件：

1. `config/.env`：填写密钥、Token、接口地址
2. `config/settings.json5`：设置 host、端口、安全策略
3. `config/ai_providers.json5`：选择默认 LLM / TTS / 图像提供方
4. `config/mcp.json5`：启用、禁用或调整 MCP 服务
5. `config/ssh.json5`：配置远程 SSH 目标（如有需要）

## 总结

`config` 目录并不是单一配置文件，而是一整套配置体系：

- `.env` 负责敏感变量与运行时密钥
- `settings.json5` 负责服务基础设置
- `ai_providers.json5` 负责 AI 提供方选择
- `mcp.json5` 负责 MCP 工具与服务注册
- `ssh*.json5` 负责远程连接
- 其他文件则负责权限控制、技能禁用和说明覆盖

如果你想手动安装或排查 Skill Pilot，优先理解这几个文件，基本就能掌握整个配置层的结构。
